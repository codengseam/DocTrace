# 极简评论系统产品方案

## 1. 产品定位

一款“只读、只写、不互动”的段注式评论系统，让读者像看古籍批注一样看别人点评，随手写下自己的批注，管理员后台审核后发布。

## 2. 用户故事

### 读者

- 作为读者，我在读某段文字时，想知道别人怎么看这一段，点击段落就能看到已审核的评论。
- 作为读者，我对某段有感想，想留下一句话，输入提交后知道“已提交，审核后会显示”。

### 管理员

- 作为管理员，我每天在 GitHub Issues 里集中查看读者提交的评论，筛选、整理、写入 JSON 文件，重新发布站点。
- 作为管理员，我不需要实时同步、后台管理面板或复杂配置，只靠 GitHub + JSON + 静态部署完成全部工作。

## 3. 核心功能清单（5 项）

1. **段落锚点**：每段文字有唯一 ID，右侧显示朱砂小徽章。
2. **看评论**：点击段落或徽章，底部抽屉浮起，展示该段已审核评论。
3. **写评论**：抽屉底部输入框提交评论，显示 toast“已提交，审核后显示”。
4. **抽屉交互**：点击遮罩、下滑、返回键均可关闭；抽屉高度 50vh，可上滑到 80vh。
5. **键盘适配**：输入框固定跟随软键盘，提交后键盘收起、抽屉保留，解决“弹起落不下去”问题。

## 4. 交互流程

### 4.1 看评论

```
[正文页]
   │
   ▼ 点击段落 / 右侧朱砂徽章
[底部抽屉浮起，高度 50vh]
   │
   ▼ 展示该段落已审核评论列表
[可上滑到 80vh / 点击遮罩 / 下滑 / 返回键关闭]
```

### 4.2 写评论

```
[底部抽屉]
   │
   ▼ 点击底部输入框
[软键盘弹起，输入框 fixed 定位在键盘上方]
   │
   ▼ 输入完成，点击提交
[toast：已提交，审核后显示]
   │
   ▼ 键盘收起，抽屉保留，输入框清空
```

### 4.3 抽屉关闭

```
[抽屉打开状态]
   │
   ├── 点击遮罩区 ──▶ 关闭
   ├── 下滑抽屉头 ──▶ 关闭
   └── 按返回键 ────▶ 关闭（prevent back-forward cache）
```

## 5. 信息架构

### 展示的数据

- 段落唯一 ID（页面内锚点）
- 已审核评论：评论内容、评论者昵称（可选）、评论类型印章、审核时间（可选）
- 评论数量徽章（段落右侧，仅显示数字，无具体人名）

### 隐藏的数据

- 未审核评论（读者提交后不可见）
- 评论者联系方式、IP、设备信息
- 评论点赞数、回复数
- 删除/编辑/举报等操作入口
- 章节级评论、全书级评论

## 6. 手机端 UX 细节

| 场景 | 处理方案 |
|------|----------|
| 输入框被键盘遮挡 | 输入框 `position: fixed; bottom: env(keyboard-inset-height, 0)`，配合 `visualViewport` 监听动态调整 |
| 提交后键盘弹起落不下去 | 提交时主动 `inputElement.blur()`，键盘收起，抽屉不关闭 |
| 段落徽章打扰阅读 | 徽章置于段落右侧，默认低透明度，hover / 点击时变朱砂色，尺寸 ≤ 24×24px |
| 抽屉高度 | 默认 50vh，可上滑展开到 80vh，下滑关闭 |
| 下滑关闭 | 抽屉头部设 drag handle，下滑超过 15% 高度即关闭，带 200ms 缓动 |
| 返回键 | 抽屉打开时按返回键仅关闭抽屉，不返回上一页（`history.pushState` + `popstate`） |
| 空状态 | 无评论时显示“此段尚无批注，留下第一笔” |

## 7. 视觉设计要点

### 整体风格：B 古籍批注风

- **主色**：米白/宣纸色 `#f7f4ed`、淡墨色 `#333333`、朱砂红 `#c04536`、赭石 `#8b5a3c`。
- **段落徽章**：右侧小圆角矩形或圆形印章，朱砂红，内为评论数量，宋体/仿宋数字。
- **评论抽屉**：仿便笺纸，背景为浅米色，顶部有撕边或折角效果，边框为淡赭石色。
- **评论字体**：正文用宋体/思源宋体，评论内容用楷体/仿宋，营造批注感。
- **评论类型印章**：每条评论左上角或右上角盖一枚小印章，如“考据”“感想”“疑问”“勘误”，印章色用朱砂或赭石。
- **输入框**：仿宣纸输入框，浅黄底、细墨边，占位符“在此写下批注…”。
- **Toast**：小印章式提示，淡入淡出，文案“已提交，审核后显示”。

## 8. 不做清单

明确砍掉的功能：

- 回复/楼中楼
- 点赞/踩/评分
- 章节评论、全书评论、专家团入口
- 用户注册/登录/头像
- 实时同步 GitHub API
- 评论后台管理面板
- 评论配置面板/主题切换
- 评论搜索、排序、筛选
- 图片/语音/视频评论
- 通知/消息中心
- 评论导出/分享

```
│   读者端    │ ──────────────────▶ │  GitHub Issues  │
│  抽屉输入框 │                     │   （暂存未审核） │
└─────────────┘                     └─────────────────┘
                                             │
                                             │ 管理员定期审核
                                             ▼
                                     ┌─────────────────┐
                                     │   审核通过      │
                                     │  写入 JSON 文件  │
                                     └─────────────────┘
                                             │
                                             │ 重新构建静态站点
                                             ▼
                                     ┌─────────────────┐
                                     │  部署到魔搭空间  │
                                     │  读者可见已审核  │
                                     └─────────────────┘
                                             │
                                             ▼
                                      ┌─────────────┐
                                      │   读者端     │
                                      │ 抽屉展示评论 │
                                      └─────────────┘
```

---

**核心原则**：评论系统是古籍的“批注旁白”，不是社交平台。读者只看、只写；管理员只审、只发。一切复杂功能都先砍掉。
